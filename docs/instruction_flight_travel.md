# Flight & Travel 데이터 수집 작업지시서 (2018-2026)

본 문서는 네이버 API를 활용하여 2018년부터 2026년까지의 항공 및 여행 트렌드 데이터를 수집하기 위한 세부 계획 및 지침을 담고 있습니다.

---

## 1. 프로젝트 목적 및 범위
- **목적**: 코로나19 이전(2018~2019), 팬데믹 기간(2020~2022), 엔데믹 이후(2023~2026)의 항공 및 여행 트렌드 변화 분석
- **수집 기간**: 2018년 1월 1일 ~ 2026년 3월 (현재 가용 시점까지)
- **주요 타겟 키워드**: 항공권, 해외여행, 국내여행, 일본여행, 동남아여행, LCC, 호텔예약 등

---

## 3. 보안 관리 지침 (.env 활용)

네이버 개발자 센터에서 발급받은 `client_id`와 `client_secret`은 코드 내에 직접 기록하지 않고 `.env` 파일을 통해 관리합니다.

### 3.1 .env 파일 설정법
프로젝트 루트에 `.env` 파일을 생성하고 아래 형식을 따릅니다.
```env
NAVER_CLIENT_ID=your_client_id_here
NAVER_CLIENT_SECRET=your_client_secret_here
```

### 3.2 파이썬 연동 계획
- `python-dotenv` 라이브러리를 사용하여 환경 변수를 로드합니다.
- `.gitignore`에 `.env`를 추가하여 외부로 유출되지 않도록 주의합니다.

## 4. API별 수집 상세 계획

### 4.1 네이버 데이터랩 (검색어 트렌드 & 쇼핑인사이트)
- **수집 항목**: '항공권', '기획전', '해외여행' 등 주요 키워드의 일간/월간 클릭 추이
- **수집 전략**: 
    - 2018~2026년의 장기 데이터를 월 단위 또는 분기 단위로 나누어 요청 (API 한도 및 데이터 정밀도 고려)
    - 성별/연령별 데이터를 수집하여 타겟 그룹별 트렌드 차이 분석

### 4.2 네이버 검색 API (뉴스 & 블로그)
- **수집 항목**: 특정 월/주차별 '항공권' 관련 뉴스 헤드라인 및 블로그 포스팅 제목
- **수집 전략**:
    - 검색 결과의 `total` 건수를 기록하여 시기별 관심도 정량화
    - 뉴스 제목의 키워드 빈도를 분석하여 이슈(유가 상승, 노선 확대 등) 파악

### 4.3 네이버 쇼핑 검색 API
- **수집 항목**: '여행용품', '캐리어' 등 여행 관련 카테고리 상품 트렌드
- **수집 전략**:
    - 최저가 및 브랜드 추이를 수집하여 쇼핑 트렌드 변화 기록

## 5. 단계별 실행 계획 (코드 미포함)

1. **환경 설정**: `venv` 활성화 및 `.env` 파일 작성
2. **키워드 선정**: 분석 대상 국가 및 여행 관련 핵심 키워드 리스트업
3. **수집 모듈 설계**: 
    - API 요청 및 응답 처리 모듈
    - API 호출 제한(Rate Limit)을 고려한 `time.sleep` 적용 로직
    - 에러 발생 시 재시도(Retry) 로직
4. **데이터 저장**: `data/raw/` 폴더 내에 `YYYYMM_trend.csv` 형태로 저장
5. **데이터 통합**: 분절된 월별 데이터를 하나의 데이터프레임으로 통합
